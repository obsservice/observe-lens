# ObserveLens PRD

## 产品概述

### 产品简介

ObserveLens 是一款面向 DEV/SRE 的 AI Native RCA 平台。基于企业可观测数据（Metrics、Logs、Traces、Events）和实体关系模型（Entity Catalog），它能够理解线上生产环境，像资深 SRE 一样自主规划调查路径、收集故障证据、推理根因，并为每一个 RCA 结论提供完整、可验证的证据链。

不同于传统依赖人工查询监控、日志和链路的故障排查方式，ObserveLens 以 Investigation（调查） 为核心，通过自然语言交互（ChatOps）的方式，将 AI 调查、证据验证和根因分析整合到统一工作台中，帮助用户快速完成线上故障定位，持续降低 MTTR（Mean Time To Recovery），减少线上故障带来的业务损失。

### 产品定位

ObserveLens 的定位不是一个 AI Chat 产品，也不是传统的可观测平台，而是一款 AI Native RCA Platform（AI 原生根因分析平台）。

它围绕一次完整的故障调查过程构建产品能力，实现：

- 自动接收线上告警和生成线上故障（Incident）。
- AI 主动规划调查流程（Investigation Workflow）。
- 自动调用监控、日志、Trace 等数据源收集证据。
- 基于多源证据推理故障根因（RCA）。
- 为每一个分析结论提供完整、可解释、可验证的证据链（Evidence Chain）。
- 支持用户通过 ChatOps 持续追问、验证和深入分析故障。

在 ObserveLens 中，AI 不只是回答问题，而是承担故障调查者（Investigator）的角色；用户也不再需要在 Grafana、日志平台、APM、CMDB 等多个系统之间反复切换，而是在统一的 Investigation Workspace 中完成从告警接入、调查分析、证据验证到根因确认的完整闭环。

### 产品要求

- 让 AI 真正理解生产环境；
- 让 AI 像资深 SRE 一样完成故障调查；
- 让每一个 RCA 都可信、可解释；

### 产品价值

ObserveLens 为企业提供以下核心价值：

- **降低 MTTR**：AI 自动完成调查规划和证据收集，显著缩短故障定位时间。
- **提升 RCA 可信度**：每一个结论均关联真实监控数据和证据来源，实现可解释、可验证。
- **统一故障调查入口**：将 Metrics、Logs、Traces、Events、Entity 等多源数据整合到统一工作台，减少上下文切换。
- **沉淀组织经验**：持续积累故障调查过程、RCA 结论和知识库，形成可复用的运维知识资产。
- **降低运维门槛**：通过自然语言交互（ChatOps）降低复杂可观测系统的使用门槛，让开发、测试和 SRE 都能够快速完成故障分析。

## 用户工作流

ObserveLens 围绕 AI 驱动的故障调查（AI Investigation） 展开，提供三条核心业务工作流，分别覆盖线上故障分析、智能巡检和实体探索三类典型运维场景。

### 1：Incident Investigation（故障调查）

当线上发生故障时，平台自动接收告警并创建 Incident，AI 主动完成故障调查、证据收集和根因分析，帮助用户快速恢复业务。

```
告警接入
    │
    ▼
Incidents（Active）
    │
    ▼
打开 Chat 调查
    │
    ▼
AI 调查
（Planning → Tool Calling → Evidence → RCA）
    │
    ▼
Observation 证据验证
    │
    ▼
确认根因
    │
    ▼
Incident Archived
```

核心价值：
- 自动接收第三方告警
- AI 主动调查故障
- RCA 全程可解释、可验证
- 降低 MTTR

### 2：Inspection（智能巡检）

用户可创建巡检任务，由平台定时执行健康检查并生成巡检报告，对发现的异常可一键进入 AI 深度分析。

```
创建巡检任务
    │
    ▼
定时执行巡检
    │
    ▼
生成巡检报告
    │
    ▼
查看报告
    │
    ▼
Ask AI
    │
    ▼
打开 Chat 深度分析
```

核心价值：
- 自动执行日常巡检
- 快速发现潜在风险
- AI 自动分析异常原因
- 巡检与 RCA 无缝衔接

### 3：Entity Exploration（实体探索）

用户可围绕任意可观测实体进行检索和拓扑分析，快速了解系统上下文，并随时发起 AI 问答完成问题定位。

```
搜索 Entity
    │
    ▼
查看实体信息
    │
    ▼
查看实体拓扑
    │
    ▼
选择关联实体
    │
    ▼
Ask AI
    │
    ▼
打开 Chat 分析
```

核心价值：
- 快速定位系统对象
- 可视化依赖关系
- 理解业务上下文
- 基于实体发起智能调查

## 设计原则

ObserveLens 围绕 **AI Investigation（智能调查）** 构建产品能力，遵循以下设计原则：
- **Investigation First**：所有功能围绕故障调查展开，统一汇聚到 Chat 工作台。
- **Evidence Driven**：所有 RCA 结论必须关联真实证据，保证结果可解释、可验证。
- **AI First**：AI 主动规划调查路径、收集证据并推理根因，用户负责验证和决策。
- **Integration First**：充分复用企业现有的 Prometheus、Grafana、Loki、Jaeger、CMDB 等平台，不重复建设指标、日志、Trace 存储、Dashboard 和告警规则等基础能力。
- **Workspace First**：Incident、Inspection、Entity 等不同业务入口最终汇聚到统一的 AI Investigation Workspace，提供一致的调查体验。

## 页面设计

### Chat

#### 页面功能

Chat 是 ObserveLens 的 AI 调查工作台，统一承载故障调查、巡检分析、实体分析及 ChatOps 等 AI 交互场景。

主要功能包括：
- 创建和管理调查会话（Session）
- 发起 AI 故障调查与自然语言交互
- 展示 AI 调查过程、工具调用及 RCA 分析结果
- 查看调查过程中关联的监控、日志、Trace 和实体拓扑等证据

#### 页面布局

页面采用三栏布局：
| 区域                 | 功能说明                                          |
| ------------------ | --------------------------------------------- |
| **Session List**   | 展示历史调查会话，支持新建、搜索、切换和管理调查会话。                   |
| **Chat Workspace** | 展示用户与 AI 的对话、调查过程、工具调用及 RCA 分析结果，输入框固定在工作区底部。 |
| **Observation**    | 展示当前调查步骤关联的监控、日志、Trace、拓扑等真实证据，并随调查步骤自动联动切换。  |

```text

|------------------------------|-------------------------------------------------------|-------------------------|
| + New Chat    🔍 Search       |                                           👤 User     |                         |
|------------------------------|   为什么 Kafka Consumer Lag 持续增长？                 |                         |
| Kafka Consumer Lag           |                                                       |                         |
| Broker CPU Analysis          | 🤖 ObserveLens                                        |                         |
| Daily Inspection             |                                                       |                         |
| Pod CrashLoop                | ✓ Step1  制定调查计划                                |                         |
| Topic Throughput             | ✓ Step2  查询 Metrics  -----------------------------> | Metrics                 |
| ...                          | ✓ Step3  查询 Logs     -----------------------------> | Logs                    |
|                              | ✓ Step4  分析 Trace    -----------------------------> | Trace                   |
|                              | ✓ Step5  根因分析                                    | Topology                |

|                              | Recommendation：Scale Broker                          |                         |
|                              |-------------------------------------------------------|                         |
|                              | Prompt Input                                          |                         |
```
